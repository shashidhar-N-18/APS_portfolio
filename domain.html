Domain: Alphabet Inc.

For this portfolio, I chose Alphabet Inc. — the parent company of Google — as my primary domain due to its vast technological footprint and multidisciplinary operations. Its ecosystem spans artificial intelligence, cloud computing, consumer electronics, digital advertising, and more, offering a rich context for applying core computing principles.

Alphabet Inc. is the parent company of Google and several former Google subsidiaries, created as part of a major restructuring of Google on October 2, 2015. Founded by Larry Page and Sergey Brin, Alphabet was designed to make the core business of Google more focused, while allowing other ambitious ventures the freedom to operate independently.

The idea was to separate Google’s traditional web services — like Search, Gmail, Maps, and YouTube — from other experimental or specialized divisions such as Waymo (self-driving cars), Verily (life sciences), DeepMind (artificial intelligence), and X (moonshot factory). This enabled better clarity, agility, and transparency both internally and for investors.

Today, Alphabet is one of the world’s most influential technology conglomerates, managing innovations across sectors such as AI, healthcare, autonomous systems, and internet infrastructure. Its structure has become a model for modern tech conglomerates aiming to balance core profitability with bold experimentation.

Google Search

Inverted Index maps words to documents for fast retrieval; Heap ranks top results by relevance. Used widely in search engines, databases, and text retrieval systems.


Where is it used?

Applications

  • Google Search: Indexes web pages for rapid query matching.
  • Elasticsearch: Powers enterprise search with inverted indices.
  • Databases: Enables full-text search in SQL/NoSQL systems.

Steps:

  1. Tokenize documents into words.
  2. Build inverted index (word → doc IDs).
  3. For query, fetch doc IDs per term.
  4. Score docs (e.g., TF-IDF).
  5. Use min-heap for top-K results.
  6. Return ranked list.
Code Snippet 

from collections import defaultdict
import heapq

class InvertedIndex:
    def __init__(self):
        self.index = defaultdict(list)
    
    def add_document(self, doc_id, text):
        for word in text.lower().split():
            self.index[word].append(doc_id)
    
    def search(self, query, k=10):
        doc_scores = defaultdict(float)
        for word in query.lower().split():
            for doc_id in self.index[word]:
                doc_scores[doc_id] += 1
        heap = [(-score, doc_id) for doc_id, score in doc_scores.items()]
        heapq.heapify(heap)
        return [heapq.heappop(heap) for _ in range(min(k, len(heap)))]

Google Ads

Greedy Algorithm optimizes ad selection; Priority Queue ranks ads by bid and quality score. Used in real-time ad auctions, online advertising platforms, and resource allocation systems.


Where is it used?

Applications

  • Google Ads: Selects and ranks ads in real-time auctions.
  • Ad Platforms: Used in Meta Ads, Amazon Ads for bid optimization.
  • Resource Allocation: Optimizes task scheduling in cloud systems.

Steps:

  1. Collect advertiser bids and quality scores.
  2. Calculate ad rank (bid × quality score).
  3. Insert ads into priority queue by rank.
  4. Greedily select top-ranked ads for slots.
  5. Charge advertisers based on second-price auction.
  6. Display selected ads to users.
Code Snippet 

import heapq

class AdAuction:
    def __init__(self):
        self.ads = []
    
    def add_ad(self, ad_id, bid, quality_score):
        rank = bid * quality_score
        heapq.heappush(self.ads, (-rank, ad_id, bid))  # Negative for max-heap
    
    def select_ads(self, k):
        top_ads = []
        for _ in range(min(k, len(self.ads))):
            rank, ad_id, bid = heapq.heappop(self.ads)
            top_ads.append((ad_id, -rank))
        return top_ads

Google Maps

A* Algorithm finds shortest paths on graphs; Graph models roads and intersections. Used in navigation, gaming, and logistics.


Where is it used?

Applications

  • Google Maps: Computes shortest routes with traffic data.
  • Navigation Apps: Waze, Apple Maps use A* for routing.
  • Gaming: Pathfinding for characters in video games.

Steps:

  1. Model roads as graph (nodes, edges).
  2. Add start node to open list.
  3. Compute f(n) = g(n) + h(n) for nodes.
  4. Select node with lowest f(n).
  5. Expand neighbors, update costs.
  6. Repeat until goal reached.
Code Snippet 

import heapq
def a_star(graph, start, goal):
    open_list = [(0, start)]
    g_score = {start: 0}
    f_score = {start: heuristic(start, goal)}
    came_from = {}
    while open_list:
        _, current = heapq.heappop(open_list)
        if current == goal:
            return reconstruct_path(came_from, goal)
        for neighbor, cost in graph[current]:
            tentative_g = g_score[current] + cost
            if tentative_g < g_score.get(neighbor, float('inf')):
                came_from[neighbor] = current
                g_score[neighbor] = tentative_g
                f_score[neighbor] = tentative_g + heuristic(neighbor, goal)
                heapq.heappush(open_list, (f_score[neighbor], neighbor))
    return None

Waymo Navigation

Dijkstra’s Algorithm finds shortest paths on weighted graphs; Graph models road networks. Used in autonomous driving, logistics, and network routing.


Where is it used?

Applications

  • Waymo: Optimizes routes for self-driving cars.
  • Logistics: UPS, FedEx use Dijkstra’s for delivery routes.
  • Networking: Routing protocols in internet infrastructure.

Steps:

  1. Model roads as weighted graph.
  2. Initialize distances, set start to 0.
  3. Add nodes to priority queue.
  4. Extract node with min distance.
  5. Update neighbor distances.
  6. Repeat until goal reached.
Code Snippet 

import heapq
def dijkstra(graph, start, goal):
    distances = {node: float('inf') for node in graph}
    distances[start] = 0
    pq = [(0, start)]
    came_from = {}
    while pq:
        dist, current = heapq.heappop(pq)
        if current == goal:
            return reconstruct_path(came_from, goal)
        for neighbor, weight in graph[current]:
            new_dist = dist + weight
            if new_dist < distances[neighbor]:
                distances[neighbor] = new_dist
                came_from[neighbor] = current
                heapq.heappush(pq, (new_dist, neighbor))
    return None

Google Assistant

Trie stores command prefixes; BFS matches user queries to commands. Used in voice assistants, autocomplete, and spell checkers.


Where is it used?

Applications

  • Google Assistant: Matches voice commands to actions.
  • Voice Assistants: Siri, Alexa use similar structures.
  • Search: Autocomplete in search bars, code editors.

Steps:

  1. Build trie with command keywords.
  2. Tokenize user voice input.
  3. Use BFS to traverse trie.
  4. Match input to command paths.
  5. Rank matches by relevance.
  6. Execute top command.
Code Snippet 

class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_end = False
        self.command = None

class Trie:
    def __init__(self):
        self.root = TrieNode()
    
    def insert(self, words, command):
        node = self.root
        for word in words:
            if word not in node.children:
                node.children[word] = TrieNode()
            node = node.children[word]
        node.is_end = True
        node.command = command
    
    def search(self, words):
        node = self.root
        queue = [(node, [])]
        while queue:
            node, path = queue.pop(0)
            if node.is_end:
                return node.command
            for word in words:
                if word in node.children:
                    queue.append((node.children[word],
                     path + [word]))
        return None
